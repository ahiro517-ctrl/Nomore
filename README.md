# 腹八分目ゲーム 🍽️

2つの料理から食べる方を選び、理想の **「腹八分目（満腹度80%）」** を目指すスマホ向けWebゲームです。
現実のカロリーコントロールと同じで、**よく膨れて低カロリーな食べ物（野菜・果物・汁物）** と
**膨れないのに高カロリーな食べ物（スナック・スイーツ・揚げ物）** をどう選ぶかが攻略の鍵。食の知識が武器になります。

## 遊び方

1. **設定画面**で性別・年齢・身長・体重・活動レベルを入力すると、Mifflin-St Jeor 式で
   基礎代謝・1日の必要カロリー・**この1食の目標カロリー**を自動計算します。
2. 2つの料理が次々と出てくるので、食べる方をタップ。どちらも嫌なら**スキップ**。
3. お腹を **腹八分目（80%）** に寄せるほど高スコア。
4. **100%を超えると食べすぎで失敗**。逆に少なすぎても物足りない結果に。**引き際が肝心！**
5. 満腹度とカロリーの**両方**がスコアに反映されます。

## 技術構成

- 純粋な HTML / CSS / JavaScript（ビルド不要・依存ライブラリなし）
- 料理データは `foods.js`、ゲームロジックは `game.js`
- 体型データ・ベストスコアは `localStorage` に保存

## ローカルで動かす

```bash
# このディレクトリで簡易サーバを起動
python3 -m http.server 8000
# ブラウザで http://localhost:8000 を開く
```

ファイルを直接 `index.html` で開いても動作します。

## GitHub Pages で公開する

`.github/workflows/deploy.yml` により、対象ブランチへ push すると自動デプロイされます。

初回のみ、リポジトリの **Settings → Pages → Build and deployment → Source** を
**「GitHub Actions」** に設定してください。以降は push するたびに公開サイトが更新されます。
